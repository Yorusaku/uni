### **Q2: 如何在项目中使用？**

#### **步骤 1：在 main.js 中初始化**

```typescript
// main.js
import { createApp } from './main' // 你已有的 createApp 函数
import { init } from '@miaoma/monitor-sdk-uniapp'

// ✅ 初始化监控 SDK（在应用启动时）

console.log('[Monitor] 监控 SDK 初始化完成 ✅')

// ✅ 创建应用实例

// ✅ 挂载应用
```

#### **步骤 2：在 HTTP 客户端中集成性能监控**

修改 `utils/http/client.ts`：

```typescript
// utils/http/client.ts
import { get as alovaGet, post as alovaPost } from 'alova'
import type { Method } from 'alova'
import { createAlova } from 'alova'
import { UniappAdapter } from '@alova/adapter-uniapp'
import { createVuexStateManager } from 'alova/state-manager'
import { Metrics, getTransport } from '@miaoma/monitor-sdk-uniapp' // ✅ 新增导入

// 创建 alova 实例
const alovaInstance = createAlova({
  baseURL: 'https://www.liuliuzhua.cn',
  timeout: 10000,
  adapter: UniappAdapter,
  stateManager: createVuexStateManager(),
  events: {
    beforeSend: (method: Method) => {
      method.requestConfig.startTime = Date.now() // 记录开始时间
    },
  },
})

// ❌ 删除 luch-request 相关代码

// ✅ 修改 get 方法（添加性能监控）
export function get<T>(url: string, params?: Record<string, any>) {
  const startTime = Date.now()
  
  return alovaInstance.get<T>(url, { params }).send()
    .then((response) => {
      // ✅ 上报接口性能数据
      reportApiResponse(url, startTime, response.status)
      return response.data
    })
    .catch((error) => {
      reportApiResponse(url, startTime, error.status)
      throw error
    })
}

// ✅ 修改 post 方法（添加性能监控）
export function post<T>(url: string, data?: any) {
  const startTime = Date.now()
  
  return alovaInstance.post<T>(url, data).send()
    .then((response) => {
      // ✅ 上报接口性能数据
      reportApiResponse(url, startTime, response.status)
      return response.data
    })
    .catch((error) => {
      reportApiResponse(url, startTime, error.status)
      throw error
    })
}

// ✅ 上报接口性能（新增函数）
function reportApiResponse(url: string, startTime: number, statusCode?: number) {
  const duration = Date.now() - startTime
  const metrics = new Metrics(getTransport())
  metrics.reportApiResponse(url, startTime, statusCode)
}
```

#### **步骤 3：配置后端接收地址**

你需要搭建一个后端服务来接收监控数据，例如：

```javascript
// server.js (Node.js)
const express = require('express')
const app = express()

app.use(express.json())

// 接收监控数据
app.post('/api/monitor', (req, res) => {
  const { events, batchSize } = req.body
  
  console.log(`[Monitor] 收到 ${batchSize} 条监控数据`)
  
  events.forEach(event => {
    if (event.event_type === 'error') {
      console.error('[Error]', event.message)
      console.error(event.stack)
    } else if (event.event_type === 'performance') {
      console.log('[Performance]', event.name, event.value, 'ms')
    }
  })
  
  res.json({ status: 'ok' })
})

app.listen(3000, () => {
  console.log('Monitor server running on port 3000')
})
```

然后在 `main.js` 中配置：
```typescript
init({
  dsn: 'http://localhost:3000/api/monitor', // 本地测试
  enableErrors: true,
  enableMetrics: true,
})
```

## 📊 **监控能力说明**

### **1. 错误监控（自动）**

监控 SDK 会自动捕获以下错误：

| 错误类型 | 说明 | 示例 |
|--------|------|------|
| **JavaScript 错误** | 运行时异常 | `undefined is not a function` |
| **Promise 错误** | 未捕获的 Promise rejection | `UnhandledPromiseRejection` |
| **网络错误** | HTTP 5xx 响应 | `500 Internal Server Error` |
| **API 失败** | uni.request 失败 | `fail: network timeout` |

### **2. 性能监控（自动）**

监控 SDK 会自动上报以下指标：

| 指标名称 | 说明 | 告警阈值 |
|--------|------|----------|
| **first-screen** | 首屏渲染时间（onLoad ~ onReady） | > 2000ms |
| **route-switch** | 路由跳转时间（navigateTo ~ onLoad） | > 800ms |
| **api-response** | 接口响应时间（超过 1秒 或 5xx） | > 1000ms |

## 🚨 **注意事项**

### **1. 生产环境配置**
```typescript
// main.js
const isDev = uni.getSystemInfoSync().platform === 'devtools'

init({
  dsn: isDev 
    ? 'http://localhost:3000/api/monitor' // 开发环境
    : 'https://your-server.com/api/monitor', // 生产环境
  enableErrors: true,
  enableMetrics: true,
})
```

### **2. 性能影响**
- ✅ 监控 SDK 本身性能开销极小（< 1ms）
- ✅ 批量上报机制（每 5秒 批量发送一次）
- ✅ 离线缓存（发送失败自动重试）

### **3. 隐私保护**
- ❌ 不上报用户隐私数据（手机号、身份证等）
- ❌ 不上报敏感接口（支付、密码修改等）
- ✅ 上报的数据都是性能指标和错误堆栈
